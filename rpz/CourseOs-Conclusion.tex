\section*{Заключение}\label{sec:conclusion}
\addcontentsline{toc}{section}{Заключение}

Взаимодействия вычислительной платы Raspberry Pi с контроллером PCD8544 осуществляется посредством SPI интерфейса.
Для реализации отправки команд и данных на микросхему с экраном необходимо реализовать драйвер символьного устройства.

При установке загружаемого модуля ядра, взаимодействующего с контроллером, требуется последовательно вызвать ряд функций:
\begin{enumerate}
\item \textit{gpio\_request()} - запрос пина \textit{GPIO}.
\item \textit{MKDEV()} - макрос, который возвращает переменную типа \textit{dev\_t}.
\item \textit{alloc\_chrdev\_region()} - выделение диапазона номеров символьного устройства. Старший и младший номера будут выбраны автоматически.
\item \textit{cdev\_init()} - инициализирует символьной устройство структурой \textit{struct fops}.
\item \textit{cdev\_add()} - добавляет символьное устройство в систему. После добавления, устройство становится доступным для вызовов.
\item \textit{class\_create()} - используется для создания указателя на структуру \textit{struct class}.
\end{enumerate}

Результатом выполнения вышеописанных функций является специальный файл устройства в каталоге \textit{/dev} и класс драйвера в \textit{/sys/class}.

Реализовано пользовательское консольное приложение, которое предоставляет следующие возможности:
\begin{itemize}
\item установить значение конкретного пикселя на экране;
\item вывести строку сообщения пользователя на указанную строку;
\item очистить экран.
\end{itemize}

В данной работ реализованы:
\begin{itemize}
\item драйвер символьного устройства PCD8544 для операционной системы Rasbpian;
\item консольное приложение для управления драйвером.
\end{itemize}

Исследованы:
\begin{itemize}
\item особенности передачи информации плате Raspberry Pi 3;
\item возможности подключения к плате Raspberry Pi 3 контроллера PCD8544.
\end{itemize}

Разработанный комплекс программного обеспечения полностью удовлетворяет требованиям технического задания.
